# 🏋️ FitMiniApp

> Мини-приложение для Telegram с тренировками, прогрессией и ролью
> тренера\
> Backend + Mini App + Admin panel - всё в одном проекте

------------------------------------------------------------------------

## ✨ Возможности

### 👤 Пользователь

-   📅 Тренировка на сегодня
-   🏋️ Логирование подходов (вес + повторения)
-   📈 Прогресс (тоннаж, максимальный вес)
-   🔔 Уведомления о тренировках
-   💳 Подписка (mock, временно скрыта в UI)

------------------------------------------------------------------------

### 🧠 Тренер / Админ

-   🧩 Конструктор программ тренировок
-   👥 Назначение программ клиентам
-   ➕ Добавление упражнений через UI
-   📊 Просмотр клиентов

------------------------------------------------------------------------

### 🔐 Админка

-   👤 Пользователи
-   💰 Платежи
-   🔔 Уведомления
-   📚 Шаблоны программ

------------------------------------------------------------------------

## 🧱 Архитектура

    Telegram Mini App (Frontend)
            ↓
       FastAPI Backend
            ↓
     PostgreSQL

------------------------------------------------------------------------

## 🛠️ Стек

-   Backend - FastAPI, SQLAlchemy, Alembic\
-   DB - PostgreSQL\
-   Frontend - Vanilla JS (Telegram Mini App)\
-   Auth - JWT + Telegram WebApp\
-   Infra - Docker / Docker Compose\
-   Rate limit - slowapi

------------------------------------------------------------------------

## 🚀 Быстрый старт

### 1. Клонирование

    git clone https://github.com/your-username/fit-mini-app.git
    cd fit-mini-app

------------------------------------------------------------------------

### 2. Создай .env

    cp .env.example .env

Для первого production-админа укажи Telegram ID:

    ADMIN_TELEGRAM_USER_IDS=123456789

------------------------------------------------------------------------

### 3. Запуск

    docker compose up --build

------------------------------------------------------------------------

### 4. Доступ

-   Mini App: http://localhost:8000/app\
-   Admin: http://localhost:8000/admin\
-   Docs: http://localhost:8000/docs

------------------------------------------------------------------------

## 🔐 Роли

  Роль    Возможности
  ------- ---------------
  User    тренировки
  Coach   программы
  Admin   полный доступ

В production все новые Telegram-пользователи создаются как клиенты. Админом
становится только пользователь из `ADMIN_TELEGRAM_USER_IDS`; дальше он может
назначать роли в панели управления.

### Demo-вход

При `ENABLE_DEV_AUTH=true` на странице `/app` появляется блок demo-входа:

-   Пользователь - `2001`
-   Тренер - `1002`
-   Админ - `1001`

В production роли не выбираются на фронте: пользователь входит через Telegram,
а права `is_coach` / `is_admin` назначаются в базе или служебным скриптом.
